Execution Order
  1. Input & Validation
    Domain parsing, IDN/punycode, devNull detection
    Sequential
  2. Parallel Protocol Collection
    DNS Records, Email Auth, DNSSEC/DANE, CT, SMTP, Policy, Registrar
    Parallel
  3. Result Assembly
    assembleResults — merge protocol outputs
    Sequential
  4. Confidence Calibration
    EWMA control charts, calibrated confidence per protocol
    Sequential
  5. ICAE/ICUAE Scoring
    Unified confidence snapshot, maturity assessment
    Sequential
  6. Drift Detection
    ComputePostureDiff, severity classification, notification queue
    Sequential
  7. Persistence & Side Effects
    saveAnalysis, storeTelemetry, currency recording, SHA3 integrity
    Async
Quality Gates
SHA-3-512 Integrity Active
Every scan's phase telemetry is hashed; tamper-evident audit trail
EWMA Control Charts Active
5 ICUAE dimensions tracked: TTL Compliance, Resolver Consensus, Source Credibility, Temporal Validity, Chain Completeness
Posture Drift Detection Active
Automatic diff between consecutive scans with severity classification (info/warning/critical)
Confidence Calibration Active
Per-protocol confidence scores (0.0-1.0) based on resolver consensus and data freshness
Ephemeral Gating Active
devNull, ephemeral, non-existent domain filtering prevents noise in the dataset
