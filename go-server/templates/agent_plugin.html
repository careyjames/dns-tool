DEVONagent Pro Plugin

Add intelligence-grade DNS security analysis to your DEVONagent Pro research workflows. Each query returns up to 15 enriched results — security reports, reconnaissance data, integrity-verified snapshots, and provenance metadata.

When a domain is analyzed through this plugin, you'll see a Plugin tag next to the domain in your analysis history.

What You Get

  • 1. Engineer's DNS Intelligence Report — full security analysis with live scanning (the engineering report)
  • 2. Observed Records Snapshot — reconstructed zone file, SHA-3-512 verified
  • 3. Detailed Security Badge — full posture score badge
  • 4. Covert Security Badge — scotopic-optimized embeddable badge
  • 5. Covert Recon Report — full analysis in covert operations mode
  • 6. Executive Intelligence Brief — board-ready executive summary
  • 7. DNS Topology — topology visualization, signal flow, and RFC source mapping
  • 8. Zenodo — Concept DOI — permanent scientific record (10.5281/zenodo.18854899)
  • 9. Full Intelligence Data (JSON) — complete analysis payload with all collected intelligence
  • 10. Discovered Domains & Subdomains — human-readable subdomain reconnaissance data (CSV)
  • 11. Sources & Methodology — RFC citations, data sources, and scoring methodology
  • 12. SHA-3 Integrity Checksum — cryptographic verification of analysis data
  • 13. Security Remediation Plan — actionable remediation steps for the domain
  • 14. Internet Archive (Wayback Machine) — permanent archived record of the analysis
  • 15. Confidence Page — SHA-3 hash audit, integrity timestamps, and analysis confidence metrics

Download & Install

Download Plugin (.plist) Download Plugin (.json)
Why two formats? Every built-in DEVONagent plugin (Google, arXiv, OpenRadar, etc.) uses Apple's native plist (Property List) format — it's the documented, canonical format with full feature support including categories, crawl delays, and XML/Atom result parsing. DEVONagent also accepts JSON files in the Plugins directory, which is simpler to read and edit. Both work identically for our plugin. The plist is recommended if you want to match the native format.

Open your preferred search set and check the box next to DNS Tool - Domain Security Intelligence. It will appear under the Security category. Query any domain name to begin.

How It Works

When DEVONagent queries a domain through this plugin, it sends a GET request to the /agent/search endpoint. Because ParseLinks is enabled, DEVONagent extracts every <a href> from the returned HTML as a separate search result — giving you direct access to every report, badge, export, and data artifact from a single query.

The __agentQuery__ placeholder in the engine URL is automatically replaced by DEVONagent with whatever domain you search for.

Plugin Anatomy

DEVONagent plugins are defined by a small set of configuration keys. Every key below is documented in DEVONagent Pro’s built-in Help (Help → DEVONagent Pro Help → Appendix → Search Plugin Format) and observable in the built-in plugins shipped at /Applications/DEVONagent.app/Contents/Resources/Plugins/.

Name
Display name shown in the plugin list and search sets. Appears in Preferences → Plugins & Scanners and in search set checkboxes.
Identifier
Unique identifier for the plugin. Convention is reverse-DNS notation or the service domain (ours: dnstool.it-help.tech).
Category
Folder group in the Plugins & Scanners window (plist only). Our plugin uses Security. Built-in categories include General, News, Science, and Shopping.
Author
Plugin author metadata (plist only). Shown in the plugin info panel. Ours: IT Help San Diego Inc.
URL
Homepage URL for the plugin (plist only). Links to this documentation page.
EngineUrl
The search endpoint. The placeholder __agentQuery__ is replaced with the user’s search term at query time. __agentOffset__ handles pagination for multi-page result sets.
ParseLinks
When true, DEVONagent extracts every <a href> from the HTML response as a separate search result. This is how our single HTML page delivers 13 distinct intelligence artifacts.
StripHTML
When false, DEVONagent preserves the HTML structure of the response (plist only). Our plugin sets this to false to retain link context and result descriptions.
ResultsPerPage
Expected number of results per page. Set to 15 to match the number of links our endpoint returns.
Keyword
Array of trigger words. DEVONagent may prioritize this plugin when user queries match these terms. We declare 13 terms covering DNS security protocols.
Operators
Bitmask for supported search operators (phrase, Boolean, wildcards, etc.). Value 59 enables the standard operator set.
UserAgent
Custom User-Agent header sent with requests (plist only). Ours identifies the plugin version in server logs: DEVONagent/3 (dnstool-plugin/1.2).
Encoding
Declares the character encoding for response parsing (plist only). Set to UTF-8.
Start / OffsetPerPage
Pagination controls. For single-page results, set Start: 0 and OffsetPerPage: 1. Multi-page plugins increment the offset per page to fetch subsequent result pages.

Plist vs JSON — Format Comparison

Status
Plist is the native format — every built-in plugin shipped with DEVONagent (Google, arXiv, Bing, Wikipedia, etc.) uses it. JSON is also supported — DEVONagent reads JSON files placed in the Plugins directory.
Category support
Plist only. Plugins appear under named folder groups (Security, Science, General, etc.) in the Plugins & Scanners window. JSON plugins appear ungrouped.
Plist-only keys
Author, URL, UserAgent, Encoding, StripHTML, and Category are recognized only in plist format. JSON plugins fall back to DEVONagent defaults for these settings.
XML/Atom parsing
Plist only. Keys like ResultXPath, TitleXPath, and DescriptionXPath allow structured XML extraction. Not applicable to our ParseLinks-based HTML plugin, but available for plugins that consume XML or Atom feeds.
Human readability
JSON is cleaner and easier to hand-edit. Plist is verbose XML but integrates with macOS tooling (plutil, defaults read, Xcode).
Our recommendation
Use the plist. It enables category grouping, custom User-Agent identification, and matches the format of every built-in plugin. We ship both formats for convenience.
